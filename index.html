Message distributing center (MDC)

This project is designed for quick establish an message distributing center to handle huge amount of notification, verification messages that come from different services and functional component.

What is Message Distributing Center (MDC)

This project is built for enterprise level message distributing to handle SMS, Email, WeChat and other notifications. There are some benefits by using this message distributing center to meets enterprise level demands, stability, standardization, scalability and extensibility.

RabbitMQ

Enterprise level product couldn't effort high lost rate of messages due to degrading of user experience and service credibility.

By using RabbitMQ, all the message should be able to temporarily queued and digest later, thus even if the consumers are too busy to handle all the notification sending task at the moment, the messages won't be lost anyway. And if the message failed to be sent by vendors, it will be requeued and resend later.

Standard Restful API

Standardizing the development procedure and components always leverage Enterprise level development efficiency and capability.

MDC are strictly following standard restful standard to provide restful API to each message features. Any internal micro-service and functional component are able to connect to MDC with standard http request to create a message sending instance and arranged into digest task queue.

Pluggable

There should be vary kinds of plugins to connect MDC to these populate SMS, Email and WeChat Vendors. The pluggable mechanism should also enable developers to implement plugins for any other vendors that is not in our list, and if you think your plugin will be favorited by others, you really should consider to publish your plugin as an open source contribution, people will love you!

Scalability

On producer side, MDC using standard node web server, thus if it is needed, you can add your own cluster and load balancers scale your service.

On consumer side, MDC are available and easy to add more consumers as much as you need, depends on your actual business scale.
